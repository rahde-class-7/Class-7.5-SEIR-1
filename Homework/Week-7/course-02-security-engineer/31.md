## IAM

What does IAM stand for <br>
IAM stands for identity and access management

What does IAM do <br>
A fine grained access control and visability for centrally managing cloud resources.  

What is access control <br>
Access control is a security technique that regulates who or what can view or use resources

SUMMARY : What does this mean <br>
IAM is the one place where an user can define who can view, access, or use resources within GCP

In this context `IAM is the one place where an user can define who can view, access, or use resources within GCP` <br>
1. Who is <br>
An identity, member, or email
2. The ability to view, access, or use resources is <br>
An role which is an collection of permissions. <br>
a permission is a specific authorization granted to an identity that defines exactly what actions they can perform on a resource, such as reading, writing, or deleting a file. <br>
3. Resources are <br>
Google resources such as compute, and BigQuery

First example of an IAM policy in simple terms <br>
An `user` can `create VM` in `Compute Engine`
1. User is an example of `who`
2. Create VM is an example of `The ability to view, access, or use resources`
3. Compute Engine is an example of an `resource`

Second example of an IAM policy in simple terms <br>
An `user` can `delete create bucket` in `Cloud Storage`
1. User is an example of `who`
2. delete create bucket is an example of `The ability to view, access, or use resources`
3. Cloud Storage is an example of an `resource`

what makes an policy binding <br>
an identity + role = policy binding

Types of cloud identity
1. Google Account - An individual identity that represents a human user (like an employee or student) and provides access to Google services through a single set of login credentials.
2. Service Account - A specialized identity used by an application or a virtual machine to interact with Google Cloud resources programmatically without requiring human user credentials. 
3. Google Workspace - A collection of cloud-based productivity and collaboration tools (such as Gmail, Drive, and Docs) designed for businesses and organizations to manage their professional communication.
4. Cloud Identity Domain - A managed container that serves as the root of your organization's identity structure, allowing you to manage users, groups, and security settings across Google services.
5. Google Groups - A collection of multiple user identities that allows you to manage permissions, mailing lists, and resource access for many people at once.
