Primitive roles

Problems with with primitive roles
1. giving users broad access. this is too much access. this broad access is at the project level. meaning you can access every resource within an project
2. does not follow the principal of least privlidge because it gives broad level of access.

3 types of primitive roles
1. owner - can read resources + perform actions on resources + manage users, groups, billing
2. editor - can read resources + perform actions on resources
3. reader - read only permissions for all reasources within an project
